# How to host your resume on Github pages

This instruction will help you get a general idea of hosting a resume on Github. You can follow this instruction to achieve serval goals:
+ Creating a Github account
+ Creating a new repository and building a README file
+ Using the basic Markdown syntax to formate your resume in Markdown version
+ Posting your resume to Github and choose a theme in setting
+ Getting the final URL links of your resume and use it whenever you want

### Prerequisites
+ a computer with internet access
+ a basic understanding on Markdown formate.(It is easy and there is helping resources in the end)

### Creating your Github account
1. Clicking the [links](https://www.github.com) here or google Github
2. Clicking on sign up and filled out the sign up form
3. Choosing a plan
```
Choosing free plan will allow you hosting resume on Github
```
4. Finishing setup by answering two multiple-choice questions
5. Verifying your e-mail address to start using Github

### Creating a new repository with a README file 
1. Clicking on + symbol on the top right corner of Github page
2. Selecting new repository
```
First, name your repository
Second, select the checkbox of "Initialize this repository with a README"
```
3. Clicking on "Create repository"
4. Editing the README filled
```
you can know see the name of your repository on the bottom of the page, Click the little pencil symbol on the top bar. It will allowed you edit the README file.
```
5. Start formatting your resume into Markdown formate.
```
The Github itself will allowed you editing online and see the preview. You can use editing tools like atom to do the formatting if you want.
```

### Formatting your resume
1. Formatting your resume to Markdown version.
```
If you heave not learned Markdown before, do not worried. It is a formate that will help your resume looks more cleaner. You can just learned few contents to formatting your resume, they are:
```
| Contents      | 
| ------------- |
| Headers       | 
| Emphasis      | 
| Lists         |
|Links          | 
|Images         |
|Syntax Highlighting|
|Tables|
|Blockquotes|
|line Breaks|

[A useful website that have all these contents](https://github.com/adam-p/markdown-here/wiki/Markdown-Cheatsheet)



2. Committing changes
```
If you use other editing tools, just copy them to the README file on Github. Otherwise, after checking the preview, you can click the green button at the bottom of the page"Commit changes"
```

### Choosing a theme for your resume
1. Clicking on "Setting" button 
2. Dragging the page to the bottom and finding the title of "Github Pages"
3. Clicking on "Choose a theme"
```
There are 12 theme that is build-in by Jekyll, you can choose one of them as your resume theme.
```
4. Clicking on "Select theme" when you find out you favorite theme.
5. Clicking on "Selecting" button again, drag the page down to "Github pages". Now you can see "Your site is ready to be published at" and a link. Clicking that link will be a preview of your final resume.
6. Fixing your resume site and make it looks gorgeous.
```
There may have some syntax error after applying a theme to your resume, fix them by changing your original README file will fix it.
```

### FAQs
Q1. Some theme showed the name of my repository as title, Can I change the name of repository?
+ Yes, you can easily changed that by Clicking "Setting". You can rename your repository whenever.

Q2. Can I changed theme?
+ yes, you can. you can always change you theme by going back to the "Setting". But, you need to wait a bit. It will not change immediately
